# 🚀 Support Pilot: Progress & Analysis Guide

This guide describes how to monitor and evaluate the performance of your **RentBasket Customer Support Bot** during the initial pilot phase.

## 1. Quick Health Check (Dashboard)
To see a summary of how the bot is doing right now, run the following command:

```bash
python3 scripts/pull_analytics.py
```

### Key Parameters to Watch:
- **AI Self-Resolution Rate**: 
    - **Goal**: > 75%
    - **What it means**: The percentage of support sessions that resulted in a ticket (handled by AI) vs. those that were escalated (handed to a human).
- **Escalation Count**: 
    - High escalation usually means either (1) users are clicking "Talk to Team" because they are frustrated, or (2) the AI hit an error/fallback.
- **Top Issue Categories**: 
    - Are people mostly asking about Maintenance or Billing? Use this to prioritize policy refinement.

## 2. Deep-Dive Metrics (Post-Pilot Analysis)

| Metric | How to Check | Ideal Target |
| :--- | :--- | :--- |
| **Intent Accuracy** | Compare `sessions.active_agent` with the `operations_tickets.issue_type`. | > 90% |
| **Turn Efficiency** | Query the `messages` table filtered by `session_id`. | < 6 turns per ticket |
| **Fallback Rate** | Count occurrences of `[SEND_HANDOFF_BUTTONS]` in `messages`. | < 10% |
| **Repeat Issues** | Look for multiple tickets from the same `phone_number` within 24 hours. | < 5% |

## 3. Important Safety Fallbacks
During the pilot, the bot is equipped with the following "Fail-Safe" logic:
1. **DB Failure**: If the ticket system goes down, the bot says: *"I'm having a temporary issue... sending to our team manually!"*
2. **LLM Failure**: If OpenAI hangs, the bot immediately triggers the **Human Handoff Buttons**.
3. **Ambiguity**: If a user's free text is unparseable, the bot provides a policy-bound summary and asks for confirmation before proceeding.

## 4. Troubleshooting the Pilot
- **Bot is not responding?**
    - Check the terminal output where `webhook_server.py` is running for Error 500 or 401.
- **Bot keeps escalating?**
    - Review `scripts/pull_analytics.py` under **Recent Escalations** to see the raw reason codes.

---
> [!IMPORTANT]
> **Weekly Review Step**: Every Friday during the pilot, pull the `operations_tickets` and verify if the *Summary* generated by the AI accurately reflects the *Description* provided by the user. If they mismatch, we need to adjust the `support_agent` system prompt.
